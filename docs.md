## Architecture Overview
The project follows a feature-based organization:
- **`lib/features/workspace`**: Orchestrates the overall screen layout and global state.
- **`lib/features/map_canvas`**: Handles map rendering, coordinate projections, and navigation controls.
- **`lib/features/layers`**: Manages the domain models and UI for map layers.
- **`lib/widgets`**: Contains reusable UI elements like menus and status bars.

## Entry Point
### `lib/main.dart`
The application initializes by:

## Main Workspace
### `lib/features/workspace/presentation/gis_workspace_screen.dart`
The `GisMainWorkspace` is the primary `StatefulWidget`. It manages:
- **Layer State**: Maintains a list of `LayerItem`s.
- **Coordinate Tracking**: Listens to hover events from the map to update the status bar.
- **Layer Creation**: Provides a modal dialog to add new Raster Tile, Feature Vector, or GeoJSON layers.
- **GeoJSON Import**: Uses `file_picker` to load local `.geojson` or `.json` files.

## Map Canvas & Navigation
### `lib/features/map_canvas/presentation/map_render_zone.dart`
The core rendering engine using `flutter_map`:
- **Base Layer**: Default Google Maps Satellite imagery.
- **Dynamic Layers**: Iterates through `activeLayers` to render markers, polylines, polygons, and GeoJSON data.
- **Interactions**:
    - **Hover**: Updates the geographic position under the cursor.
    - **Context Menu**: Right-click (secondary tap) opens a menu to copy current Lat/Long coordinates to the clipboard.

### `lib/features/map_canvas/presentation/components/navigation_cluster.dart`
A floating control card providing:
- **Zoom In/Out**: Smoothly adjusts map scale using `AnimatedMapController`.
- **Recenter**: Animates the map back to `(0,0)` at zoom level 2.

## Data Models
### `lib/features/layers/domain/layer_model.dart`
The system uses a polymorphic model hierarchy for layers:
- **`LayerItem`**: The base class containing common properties like `id`, `name`, `type`, and `isVisible`.
- **`TileLayerItem`**: For raster tile providers (XYZ). Includes a pre-configured `TileLayer` with disk caching (1GB limit).
- **`GeojsonLayerItem`**: Holds a `GeoJsonLayer` for rendering vector data from files.
- **`FeatureLayerItem`**: Manages manual collections of `Marker`, `Polyline`, and `Polygon` objects.

## Technical Implementation Details

### State Management
The application currently uses a **Lifted State** pattern with `setState` in the `GisMainWorkspace` widget.
- The list of `_layers` is the "source of truth".
- Callbacks like `onLayersChanged` are passed down to children (like `LayersSidebarPanel`) to trigger UI updates when data changes.

### Coordinate Projections & Precision
- The map operates in **EPSG:4326 (WGS 84)**.
- Precision: Geographic coordinates are rounded to **5 decimal places** for display in the status bar and context menus, providing approximately 1.1 meters of accuracy at the equator.
- The `MouseRegion` in `MapCanvasRenderZone` uses `camera.screenOffsetToLatLng` to perform real-time projection from screen pixels to geographic points.

### Event Interception
A `RepaintBoundary` is used around the map canvas to optimize performance during hover events, ensuring that the heavy map rendering doesn't rebuild unnecessarily when only the coordinate text in the status bar changes.

### File Handling
GeoJSON data is loaded via the `GeoJsonLayer.file` constructor, which reads local filesystem nodes provided by the `file_picker` package. This allows for fast, local-first vector visualization.

## Extension Guide

### Adding a New Layer Type
To introduce a new layer source (e.g., WFS or a custom database):
1. **Define the Model**: Extend `LayerItem` in `layer_model.dart`.
2. **Update UI Logic**: Add a new option in `_showCreateLayerModal` within `gis_workspace_screen.dart`.
3. **Implement Rendering**: Update the `layers` loop in `map_render_zone.dart` to handle the new class type.

### Modifying the Theme
The application colors are centralized in `lib/main.dart` within the `ThemeData` configuration. To change the "brand" colors, update the `scaffoldBackgroundColor`, `cardColor`, and `dividerColor`.
